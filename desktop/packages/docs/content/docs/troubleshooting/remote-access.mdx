# Remote Access

Remote AX Code access is intentionally unavailable. Remote hosts, SSH instances, LAN binding, reverse proxies, and public tunnels are rejected rather than diagnosed or retried.

If a legacy configuration points to a remote instance, restart AX Code Desktop. Startup migration removes the obsolete endpoint and stored credentials, then returns the app to its local instance. If startup reports an active legacy public tunnel, run `ax-code-desktop tunnel stop` first.

For local startup problems, see [AX Code Connection](/troubleshooting/ax-code-connection/) and [Local-only Security](/security/).
