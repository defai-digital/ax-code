# Providers, Models & Agents

Before AX Code Desktop can do anything, it needs at least one AI provider connected. This page covers connecting a provider, choosing a model, and tailoring agents.

AX Code supports Cloud API providers, local CLI provider bridges, and AX Engine local models. For the exact provider ids, credential variables, and model ids, see [Supported Providers and Models](https://github.com/defai-digital/ax-code/blob/main/docs/supported-providers.md).

## Connect a provider

1. Open **Settings → Providers**.
2. Open the **Add provider** menu and pick a provider that isn't connected yet.
3. Sign in one of two ways, depending on the provider:
   - **API key** — paste your key and save.
   - **Sign-in (device flow)** — AX Code Desktop shows a link and a short code. Open the link, enter the code, and approve. AX Code Desktop finishes connecting on its own.

When a provider shows as connected, its models become available in chat.

To disconnect, open the provider and choose to remove its sign-in.

## Local AX Engine provider

AX Engine is available only on eligible Apple Silicon Macs. It does not run as a local provider in Windows Desktop builds.

On macOS, connect AX Engine from **Settings -> Providers** after the AX Code CLI reports the local provider as available. On Windows, use hosted providers, an OpenAI-compatible gateway, or connect AX Code Desktop to a remote AX Code server running on a supported Mac if you need AX Engine.

The built-in AX Engine provider exposes `qwen3.6-27b-6bit`, `qwen3.6-35b-a3b`, `gemma-4-12b`, `gemma-4-26b`, `gemma-4-31b`, and `glm-4.7-flash`. The default is `qwen3.6-27b-6bit`.

## Pick a model

You pick the model where you're working:

- in chat, use the model picker in the message bar to set the provider and model for that session
- per agent, set a default model (below)

## Set up agents

An agent is a named setup — a model, a personality, and what it's allowed to do.

1. Open **Settings → Agents**.
2. Pick an agent, or create a new one.
3. Edit any of:
   - **description** — what the agent is for
   - **model** — its default model
   - **temperature** — how creative its answers are
   - **prompt** — standing instructions it always follows
   - **tool rules** — which tools it may use

## Where your sign-ins live

Provider sign-ins are stored by AX Code, not AX Code Desktop, so they're shared with the AX Code CLI. If you set the same provider in more than one place, the most specific setting wins: a per-project setting overrides your personal one.
