Help with the OpenWiki repository wiki for this project (path: ${path}).

OpenWiki is the **semantic** wiki layer (architecture, module intent). It does **not** replace `code_intelligence` / `lsp` for precise symbols. Guide: docs/wiki.md

User request: $ARGUMENTS

Parse $ARGUMENTS as an optional subcommand (case-insensitive):

| Arg | Action |
|-----|--------|
| empty / help | Summarize `ax-code wiki doctor` guidance and recommend next steps |
| status | Run or describe `ax-code wiki status` |
| doctor | Run or describe `ax-code wiki doctor` |
| lint | Run `ax-code wiki lint` (stale vs HEAD, index, symbol links) |
| cards | Run `ax-code wiki cards` (write `.ax-code/wiki-cards.md`) |
| related \<Symbol\> | Run `ax-code wiki related <Symbol>` |
| ensure / ensure-agents | Run `ax-code wiki ensure-agents` |
| generate / init | Run `ax-code wiki generate` (long LLM job; stream progress) |
| update / refresh | Run `ax-code wiki update` |
| otherwise | Treat as a free-form question: read `openwiki/quickstart.md` or `.ax-code/wiki-cards.md` first, then answer |

Rules:

1. Prefer the real CLI (`ax-code wiki …`) over re-implementing OpenWiki inside the session.
2. If OpenWiki CLI is missing, tell the user: `npm install -g openwiki` and configure `~/.openwiki/.env`.
3. After generate/update, ensure AGENTS markers: `ax-code wiki ensure-agents`.
4. Keep AGENTS.md thin — never paste the whole wiki into it.
5. For architecture Q&A, read the wiki index then drill down; for callers/callees use `code_intelligence`.
