## Task fit

- **council**: code quality findings, security, architecture trade-offs, “good vs worse” ratings with consensus/majority/singleton.
- **arena plan**: compare *approaches* only (not a findings review).
- **arena implement**: multi-model competing *patches* in worktrees.

## Instructions

1. Gather **brief** context (targeted read/grep). Avoid long monorepo tours before fan-out.
2. Call **council** with `kind: review` (or `design` for architecture trade-offs). Optional `debateRounds: 1` for high stakes.
3. Config is re-read each call. If disabled (`modes.council.enabled: false`), re-enable in `ax-code.json` and re-call — no session restart required.
4. Needs connected providers with selectable models; aim for ≥2 for consensus tiers. If insufficient, stop and ask the user to connect providers — do not invent multi-model agreement.
5. Summarize **consensus / majority / singleton** clearly. Advisory only — do not apply patches solely because models agreed; verify with tests when changing code.
