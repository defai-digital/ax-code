Create a bounded repair brief from an existing VerificationEnvelope in this session.

Use this after verify_project or refactor_apply has produced a failed envelope with structuredFailures, and you want a focused repair prompt that can be used by a human, the current agent, or a future bounded repair loop.

Input:
- envelopeId: 16-char hex id returned by verify_project or refactor_apply.
- maxFailures: optional cap for structured failures eligible for repair handoff. Defaults to the shared repair-handoff policy.

Output:
- decision: whether the envelope is eligible for bounded repair handoff.
- reasoning: why it is eligible or rejected.
- brief: when eligible, an anchor-first repair brief with the structured failures and first raw output lines.
- verificationRun: the source tool/callID that emitted the envelope.

Rules:
- Do not fabricate envelope ids. The id must already exist in this session.
- This tool does not edit files, run commands, spawn tasks, or mark a debug hypothesis resolved.
- If the decision is rejected, inspect the reasoning and either run a more specific verification command or handle the issue manually.
