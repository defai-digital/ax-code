Apply a VerificationEnvelope to an existing runtime debug hypothesis.

Use this after a fix attempt has produced verification metadata through verify_project or refactor_apply. The tool evaluates the whole VerificationEnvelope set from the tool.result that emitted the cited envelope, then updates the existing hypothesis in the session:

- passed verification set -> confirmed
- any failed envelope with structured failures -> refuted
- skipped-only, timeout, error, or failed without structured failures -> unchanged / inconclusive

Inputs:
- hypothesisId: 16-char hex id returned by debug_propose_hypothesis.
- envelopeId: 16-char hex id returned by verify_project/refactor_apply metadata. This selects the originating verification set; do not cite a lone passing check from a run where another check failed.

Rules:
- Do not use fabricated ids. Both the hypothesis and envelope must already exist in this session.
- Do not call this before running verification. First run verify_project or refactor_apply and cite the resulting envelopeId.
- This tool does not execute tests; it only records how an existing verification result affects a debug hypothesis.
