# Contributing to AX Code

AX Code is open source and we welcome community involvement. Development is currently led by DEFAI Digital, but there are several ways to participate.

## Sharing Your Modifications

AX Code is licensed under MIT. If you publish forks or redistributions, preserve the copyright and permission notice. We still encourage sharing improvements back. The easiest ways:

- **Open an issue** describing your change with a diff or patch
- **Publish your fork** so other users can build on your improvements
- **Write about it** — blog posts, tutorials, or discussions help the whole community

We use MIT because low-friction adoption and broad reuse help the project grow.

## Reporting Bugs

Found something broken? [Open a bug report](https://github.com/defai-digital/ax-code/issues/new?template=bug_report.yml).

Include:
- What you expected vs. what happened
- Steps to reproduce
- Your platform (OS, AX Code version, provider/model)
- Error messages or screenshots if applicable

## Requesting Features

Have an idea? [Open a feature request](https://github.com/defai-digital/ax-code/issues/new?template=feature_request.yml).

Describe the problem you're trying to solve, not just the solution you want. This helps us design the right approach.

## Questions

Stuck or confused? [Open a question](https://github.com/defai-digital/ax-code/issues/new?template=question.yml) or join us on [Discord](https://discord.gg/cTavsMgu).

## Issue Guidelines

- Use one of the provided issue templates (bug report, feature request, or question)
- Be specific and concise
- One issue per report — don't bundle multiple bugs or requests
- Check [existing issues](https://github.com/defai-digital/ax-code/issues) before opening a duplicate

> **Note:** We do not accept AI-generated issues. Issues that are clearly AI-generated walls of text will be closed without review.

## Pull Requests

We are not accepting external pull requests at this time. If you've found a bug and know the fix, please describe it in a bug report — include the fix approach and we'll handle the implementation. This may change in the future as the project matures.
